IDE

Stand: März 2013

  • LiteIDE: ist irgendwie lame: sieht lame aus, hat keine gute CodeCompletion, GOPATH unter Windows funzt irgendwie nicht, die Doku ist eine Katastrophe. Macht keinen Spaß.
  • GoClipse: Eclipse halt, CodeCompletion ist ganz OK, Build ist komplex, hat keine tollen Features, die den Einsatz rechtfertigen würden.
  • vim und emacs: nicht getestet, bisher soll alles auf Linux und Windows gehen, bin auch kein Fan von mauslosen Editoren.
  • IntelliJ IDEA Community Edition: funzt, ohne das man was einstellen muß, CodeCompletion ist ziemlich gut. Hat so nette Sachen wie package per Alt+Enter importieren, wenn man es das erste mal nutzt. Dauert lange bis es mal startet. Man sieht leicht die Doku von Methoden (mit STRG+Q) Eigentlich eine super Sache.
  • GoSublime: CodeCompletion ein kleinen ticken besser als IntelliJ, weil der noch die Parameter bei Methodenaufrufen reinschreibt, funktioniert mit gocode, ansonsten halt Sublime Text 2. Man sieht die Doku von Methoden in einem eigenen Tab (mit STRG+.,STRG+h oder SHIFT+STRG Rechtsklick), ist irgendwie eine gute Idee.

Endergebnis: GoSublime und IntelliJ liegen bisher gleichauf mit leichtem Vorsprung für GoSublime weil schneller startet, CodeCompletion ein bißchen besser ist.
